# Deployment Contract

## What the Kit Gives You

- reusable templates
- scripts and cron definitions
- reference architecture
- deployment prerequisites
- activation path
- pattern library assets
- memory scaffolding
- security and compliance docs

## What the Buyer Must Provide

- infrastructure
- system access and API credentials
- internal operator
- domain knowledge for calibration
- time for review, tuning, and rollout

## What Is Reasonable to Expect

### Day 1
- infrastructure selected
- shared brain scaffolded
- MCP reachable
- first agent selected

### Day 7
- first production workflow live
- monitoring enabled
- human review path active

### Day 30
- serious first deployment
- more than one useful workflow in production
- documented capability authority, rollback and promotion evidence
- team-level understanding of what stays human

## What Is Not Included by Default

- custom engineering for your exact stack
- vendor support coordination
- infrastructure provisioning done for you
- guaranteed autonomy or ROI rates
- ongoing managed optimization

## Support Boundary

The kit is documentation plus assets. Managed / consulting begins where your deployment requires direct implementation help.
