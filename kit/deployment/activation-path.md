# Activation Path

## Day 1

- read README.md
- complete prerequisites
- choose canonical architecture
- stand up MCP endpoint
- create the shared brain structure
- decide first workflow and first agent

## Day 7

- first real workflow in production
- logging and alerts live
- escalation rules tested
- shadow mode outputs reviewed
- first fixes documented back into the brain

## Day 30

- multi-workflow deployment
- capability authority and review paths tested
- monitoring stable
- team can operate the stack without founder babysitting every step
- clear backlog for what to automate next versus what should remain human

## Rule

Do not chase breadth first. Deploy one workflow, make it reliable, then expand.
